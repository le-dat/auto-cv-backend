# DevOps & Platform Engineering Skills

## Containerization & Orchestration
- **Docker** — container runtime; standard for packaging applications
- **Kubernetes (K8s)** — container orchestration; scaling, rolling updates, self-healing
- **Helm** — Kubernetes package manager

## CI/CD
- **GitHub Actions** — CI/CD built into GitHub
- **GitLab CI** — CI/CD with GitLab
- **Jenkins** — traditional CI/CD server
- **ArgoCD** — GitOps continuous delivery for Kubernetes

## Infrastructure as Code
- **Terraform** — declarative IaC; cloud-agnostic
- **Pulumi** — code-as-IaC using familiar languages
- **AWS CloudFormation** — native AWS IaC

## Cloud Platforms
- **AWS** — EC2, ECS, EKS, Lambda, S3, RDS, CloudFront
- **Google Cloud Platform** — GKE, Cloud Run, BigQuery
- **Azure** — AKS, Azure Functions, Azure DevOps

## Observability
- **Prometheus / Grafana** — metrics collection and visualization
- **ELK Stack** — centralized logging (Elasticsearch, Logstash, Kibana)
- **Datadog / New Relic** — APM tools
- **OpenTelemetry** — vendor-neutral telemetry collection

## Synonyms & equivalences
- K8s = Kubernetes
- GCP = Google Cloud Platform = Google Cloud
- IaC = Infrastructure as Code
- CI/CD = Continuous Integration / Continuous Delivery
